# The Problem

Most BMS operators already provide remote access in some form.

But it usually looks like this:

* Shared VPNs across multiple customers
* Static firewall rules that never get reviewed
* Credentials emailed around engineering teams
* No clear billing model
* Remote access provided “for free”

Meanwhile, remote access is now **essential infrastructure** for modern building operations.

It powers:

* remote troubleshooting
* alarm response
* energy optimisation
* system upgrades
* out-of-hours support

Yet for many operators it remains an **unmanaged cost**, not a product.
